Hexo Cheatsheet

Command Usage
hexo init [site-name] Initialize a new Hexo site named [$site-name]
hexo server Start the Hexo server on default port 4000
hexo new [post-name] Create a new post named [post-name]
hexo clean Delete database and /public folder
hexo generate or hexo g Generate the new static Hexo site under /public folder
hexo deploy or hexo d Deploy the Hexo site based on the deployment method configured in _config.yml
hexo g -d Generate & deploy
hexo new page "categories" Create a ‘/categories’ folder under ‘/source’. After adding type: "categories" in the Front-matter of /categories/index.md, the content of categorids/index.md will be shown in your-hexo-site-url/categories.
{ % post_link post-name ClickToSeeThisPost % } Inside one post, link to another post. By clicking “ClickToSeeThisPost”, will redirect to prefix-post-name.md (prefix is the prefix added to the post named based on the naming rule you set). If "ClickToSeeThisPost" is not set, the text will be shown as the post name by default.
Create drafts (reference) Use hexo new draft "new draft", or move a post into source/_draft/ folder.
Preview drafts Start the server by hexo server --drafts, then the drafts can be previewed. However if using hexo server, you won’t see the drafts.
Insert Video Refer to Tag Plugins: use { % youtube video_id % } or { % vimeo video_id % }
