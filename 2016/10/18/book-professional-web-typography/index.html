Delivering Web Fonts

How to setup the fonts in browser

  • @font-face
    • Set the font in css; have to specify the font file’s URL; need to consider browser compatibility and font optimization(which font to load first).
    • Example and explanation
  • Web Font Services
    Take care of font optimization, browser compatibility, reliable performance, and licensing agreements behind the scenes.
    1. Google Fonts
      By adding one line of <link> element in HTML.
    2. Typekit
      By adding JavaScripts to specify typekit font’s js file.
  • Web Font Loader
    • Web Font Services are convinient for web font delivery, but not the ideal methods. Problem: the fonts are referenced in the head of the page. Chrome, Firefox, Opera, and Safari, text won’t render the page until custom fonts are loaded, resulting into flash of invisible text (FOIT). Internet Explorer displays the fallback type first and then switches to custom type once the font assets finish loading, resulting in a glaring flash of unstyled text (FOUT).
    • Web Font Loader provides an optimal experience across modern browsers. It loads fonts asynchronously and cache fonts in browser. Users may see FOUT the first time they visit the site, but their experience won’t be interrupted once the font assets are downloaded and cached.
    • Example and explanation
  • Font Face Observer
    • Web Font Loader solves the FOIT issue, but doesn’t handle the FOUT issue.
    • Font Face Observer can be referenced at the end of document to avoid the extra overhead. Uses JavaScript’s scroll events to load and monitor web fonts.

Selecting Body Text

Basic font features need to be considered when choosing font for body text

  • Generous X-height
    X-height is an indicator of the sizes’ harmony between lower and upper case letters. It is the vertical measure of a lowercase x, which is usually bigger than half of the cap height(the distance between the baseline and the top of the capital letter.). The shorter the x-height, the smaller the letter will appear, which can cause the type to become illegible. It can neither be too large, as n and h will be difficult to distinguish.
  • Open Apertures
    Apertures are the opening gaps found in certain letters such as c, e, and s. Closed apetures are bad for legibility. c can be mistaken for o at smaller sizes.
  • Even Spacing
    Letters should not be too close to or too far away from each other; good for readability.
  • Clear Terminals
    Terminals are the end of the letters a, c, f, j, r, y. Distinguishable terminals make the letters more readable.
  • Distinguishable Ascenders and Descenders
    Ascenders refers to the strokes that extend above the mean line and descenders refer to the strokes that extend below the baseline. The more distinguishable the ascenders and descenders are, the more legible the letters.
  • Styles & Weights
    For a family of type to work as body text, there need to be at least 4 basic styles: regular, italic, bold, and bold italic.
  • Contrast
    Contrast refers to how different between the thick and thin strokes of a letter. High contract letters are good for posters and advertising, but might not be legible enough for long-term reading.

Choosing Headings

How to choose suitable typefaces or customize the sizes for headings

  • It’s recommended to use the same typeface for headings and body text for visual consistency and better browser loading performance.
  • If different typefaces must be mixed, it’s not recommended to use two typefaces that are fundamentally similar but still different (e.g. two serifs or two sans-serifs). It’s recommended to conbine typefaces in a same superfamily—typefaces that have different classes (e.g. sans, serif, slab) but share similar structure. Another suggestion is using typefaces from the same designer.
  • Recommend to use em but not px.
  • Modular scale: when customizing h1-h6, what should the sizes be? Use this Modular Scale tool to calculate the size of each heading scale.
